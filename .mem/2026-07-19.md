## done
- [blur] Added `-b/--blur SIGMA` Gaussian pre-blur to threshold + adaptive. New shared module src/image_wormhole/preprocess.py: add_blur_arg() registers flag; apply_blur(gray,sigma) → sigma<=0 no-op, else cv2.GaussianBlur(gray,(0,0),sigmaX=sigma); blur_tag(sigma) → "_g<sigma>" with '.'→'p' (e.g. _g2p5), "" when off. Blurred variants get _g<sigma> suffix so runs never overwrite.
- [cli] adaptive block short flag moved -b → -k (long --block unchanged) to free -b for global blur flag. BREAKING: old `iw adaptive -b 7` now means blur; use `-k 7` for block.
- [threshold] threshold now AUTO-EXTRACTS by default — per threshold value writes a SINGLE transparent RGBA matte (_black) into <stem>/threshold/; opaque binary output dropped. Added `--keep black|white|both` flag (default black). Rationale: black & white mattes are EXACT alpha inverses (alpha_black == 255 - alpha_white), perfectly partitioning image; emitting both doubled output for no editor gain (one-click invert in Photoshop/Affinity recolours trivially).
- [extract] standalone `extract` CLI command REMOVED. RGBA logic moved to new shared module src/image_wormhole/extract.py: to_rgba(gray,keep), SIDES=("black","white"), MIDPOINT=128. ops/extract.py deleted.
- [output-layout] `iw/` root prefix REMOVED from output paths. Now <source-stem>/<technique>/<stem>_<tag>.png relative to cwd. Defaults changed "iw"→"." in paths.py (variant_path/technique_dir) and both ops' --out.
- [naming] threshold filename convention now <stem>_t###_<side>[_g<sigma>].png (side ∈ black|white).
